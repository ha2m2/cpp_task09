# cpp_task09

# Unreal Engine Multiplayer Number Baseball

**언리얼 엔진 C++ 기반 클라이언트-서버 멀티플레이어 숫자 야구 게임**

## 프로젝트 개요

본 프로젝트는 언리얼 엔진의 멀티플레이어 네트워크 프레임워크(RPC, GameMode, PlayerState)를 활용하여 구현한 **숫자 야구 게임**입니다.
클라이언트의 입력은 Server RPC를 통해 서버로 안전하게 전달되며, 서버는 판정 알고리즘을 거쳐 개별 클라이언트에게 Client RPC로 정확한 피드백과 UI 업데이트를 지시합니다.

## 주요 기능

- **네트워크 동기화 및 RPC 통신**:
  - `Server RPC`를 통한 플레이어 입력 수신 및 유효성 검증.
  - `Client RPC`를 통한 개별 플레이어 UI 피드백 및 결과 공지.
- **판정 시스템**:
  - `GameMode`에서 정답 생성, 입력값 검증(길이, 범위, 중복), 스트라이크/볼 판정, 전체 진행 상태(무승부/리셋) 통제.
- **독립적인 플레이어 상태 관리**:
  - `PlayerState`를 활용하여 접속자별 시도 횟수(`CurrentGuessCount`)를 독립적으로 관리하고 네트워크에 복제(Replication).
- **동적 UI 데이터 바인딩**:
  - `UUserWidget` 상속 C++ 클래스를 통해 UMG 에디터의 `TextBlock` 및 `EditableTextBox`와 코드 단의 실시간 바인딩 구현.

## 핵심 클래스 구조

| 클래스명                   | 역할 및 설명                                                                                                                                                                      |
| :------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`BGGameModeBase`**       | 게임의 마스터. 정답(Secret Number) 생성, S/B 판정, 플레이어 접속(`PostLogin`) 관리, 게임 종료 및 리셋(`BroadcastResultAndReset`)을 담당합니다.                                    |
| **`BGPlayerController`**   | 서버-클라이언트 간의 통신 중계기. 로컬 화면에 UI를 띄우고, 사용자의 텍스트 입력을 서버로 전송(`ServerProcessGuess`)하며, 서버의 공지 명령(`ClientShowNotification`)을 수행합니다. |
| **`BGPlayerState`**        | 플레이어의 데이터 저장소. 각 플레이어의 이름(Player1, Player2)과 현재 시도 횟수를 저장하고 관리합니다.                                                                            |
| **`BGChatInput`**          | 숫자 입력을 받는 C++ UI 클래스. 엔터(Enter) 키 입력 이벤트를 감지하여 PlayerController로 전달합니다.                                                                              |
| **`BGNotificationWidget`** | 게임 결과 및 알림 텍스트를 화면 중앙에 출력하는 C++ UI 클래스.                                                                                                                    |

## 게임 규칙 (Game Rules)

1. **입력 조건**: 1에서 9 사이의 **중복되지 않은 3자리 숫자**를 입력해야 합니다. (예: `123`, `845`)
   - _조건 위반 시 기회를 소진하지 않고 경고 메시지가 출력됩니다._
2. **판정 기준**:
   - **Strike (S)**: 숫자와 자리가 모두 일치할 경우.
   - **Ball (B)**: 숫자는 일치하나 자리가 다를 경우.
   - **OUT**: 일치하는 숫자가 하나도 없을 경우.
3. **승패 조건**:
   - **승리**: 3 Strike (정답) 달성 시 해당 플레이어 승리 및 게임 리셋.
   - **대기**: 플레이어당 주어지는 **3번의 기회**를 모두 소진할 경우, 다른 플레이어의 턴을 대기.
   - **무승부**: 접속한 모든 플레이어가 기회를 소진할 경우 무승부 선언 및 게임 리셋.

## 실행 및 테스트 방법 (How to Play)

1. 프로젝트를 Visual Studio 또는 Rider에서 빌드(`Ctrl + Shift + B`)합니다.
2. 언리얼 에디터를 실행합니다.
3. 우측 하단 Play 버튼 옆의 더보기(⋮)를 클릭하고 **Net Mode**를 `Play As Listen Server`로 설정합니다.
4. **Number of Players**를 `2`로 설정합니다.
5. Play(PIE) 버튼을 눌러 두 개의 클라이언트 창에서 각각 숫자를 입력하며 멀티플레이어 동기화를 확인합니다.
